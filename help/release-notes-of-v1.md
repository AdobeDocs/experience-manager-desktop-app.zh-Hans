## 系统要求和先决条件 {#system-requirements-and-prerequisites}

AEM Desktop与以下操作系统兼容：

* Mac OS X 10.10 或更高版本（带有最新的错误修复）。
* Windows 7 和 Windows 10（带有最新的 Service Pack 和错误修复）。

Adobe强烈建议使用最新版AEM桌面应用程序来使用最新功能、最新错误修复和尽可能最佳的性能。

您计划在本地计算机上安装的AEM桌面应用程序版本需要特定的AEM服务器版本／其他服务器端组件（服务包、热修复或功能包）。 在首次连接到AEM服务器之前，请确保正确配置AEM服务器。 如果需要帮助，请与AEM管理员联系。

请参阅 [本文档结尾的详细兼容性列表](#compatibilitymatrix) ，以评估设置的先决条件。

## What&#39;s New in AEM desktop app 1.10 {#what-s-new-in-aem-desktop-app}

AEM桌面应用程序1.10侧重于在使用链接文件（如InDesign）打开资产时改进大型上传的用户体验、有关后台操作的信息以及优化的体验。

>[!NOTE]
>
>如果您使用的是macOS 10.15.4或更高版本，请至少使用应用程序版本1.10.0.6。 此修补程序版本符合 [Apple公证要求](https://developer.apple.com/news/?id=04102019a)。

**本地编辑／注销**:可以在状态窗口中禁用自动上传保存到资产的更改。 这样，用户可以继续处理文件并保存更改，然后在他们准备就绪后，决定上传所有更改。

**简化的资产状态窗口**:状态窗口已简化*上载选项卡现在显示单个资产以及文件夹／批量上载。 之前的“批量上传”选项卡已被删除。

**用于指示批量上传的应用程序图标**:应用程序图标将通过显示“转移”叠加来指示正在进行批量上传。

**更新冲突通知**:当应用程序在尝试更新资产时检测到冲突时，它将显示通知，以便用户无需监视状态窗口即可进行查看。 当应用程序开始时，它将检查所有冲突，以便用户能够解决这些冲突。

**更好地处理连接损失**:如果连接丢失，将暂停批量上传，用户稍后将能够恢复。 “重试”按钮可用于重试单个文件的上传失败。

链接警告注意事项：Creative Cloud应用程序（如InDesign）在放置物品时对上次修改时间进行“快照”。 如果该日期在以后的某个时间发生更改，Adobe Creative Cloud应用程序将报告这些链接已过时。 这通过以下几种方式进行报告：

* 启动Adobe Creative Cloud应用程序时，将显示一个对话框，通知用户链接的资产已过时，并提示用户采取操作。
* 如果Adobe Creative Cloud应用程序已在运行，则链接的资产上将显示一个黄色的三角形警告图标。

此行为与本地磁盘上的资产和AEM Desktop安装目录中的资产相同，但以下情况除外：

* 如果已置入的资产由其他用户修改，则当其他用户第一次打开包含已置入资产的文档时，将显示警告图标。 仅当已本地缓存置入的资产时，才会发生这种情况。
* 如果用户通过AEM Desktop的安装目录修改置入的资产，然后清除其本地缓存，则置入的资产将报告为过期资产。

这两种情况都是AEM Desktop的“延迟同步”架构的副作用。

## 更改列表 {#list-of-changes}

### Mac上的安装点 {#mount-point-on-mac}

自MacOS 10.12(Sierra)以来，Apple已将用于安装网络驱动器和设备的/Volumes文件夹的权限更改为限制性更强。 在那里创建新的装载点需要管理权限。 此问题已在MacOS 10.12.5中修复。

由于AEM桌面应用程序应为在本地计算机上没有管理员权限的用户运行，因此AEM资产存储库的装载点在1.4和1.5中已更改为MacOS上用户的本地文件夹中的DAM子文件夹(CQ-104183)。

由于/Volumes文件夹不再需要管理权限，因此此更改已在1.5.1中恢复。这还使得在MacOS用户之间共享已放置AEM资源的InDesign文档成为可能。

### 协议更改（自v1.3起） {#protocol-change-since}

* Mac OS X:
   * OS X桌面集成的本地网络驱动器协议从WebDAV更改为SMB1。
   * 随桌面应用程序一起安装的AEM存储库在Finder中将显示为“smb”网络驱动器，而不是WebDAV驱动器。
* Windows:
   * Windows桌面集成的本地网络驱动器协议保持不变；AEM将作为WebDAV共享进行装载。
* 对于两个平台（Windows和Mac）:
   * 用于访问／下载资产以及将更改上传到AEM的协议已更改为原生AEM协议，该协议是基于HTTP的RESTful协议。 它提供对网络操作的更大控制，并且与网络基础结构更兼容。

>[!NOTE]
>
>在Mac OS X上，本地网络驱动器协议从WebDAV更改为SMB1会导致存储库中同一资源的不同本地路径。 这可能会通过“置入”命令影响到Adobe Creative Cloud应用程序中放置的文件的链接。 有关详细 [信息，请参阅使用AEM Desktop应用程序](use-app-v1.md) 。

### 文件处理（自1.3起） {#file-handling-since}

* 文件夹会在预定义的延迟（当前为30秒）后自动更新。
* 其他用户签出的文件被标记为只读。
* 文件会分两个阶段保存到通过桌面应用程序装载的网络驱动器位置。
* 在第一阶段，文件保存在本地。 这样，保存文件的用户无需等到文件完全传输到AEM后，即可在保存文件后立即恢复工作。
* 在第二阶段，桌面应用程序在预定义的延迟（例如，30秒）后将更新的文件上载到AEM服务器。 此操作在后台进行。 使用“ **显示后台文件同步状态** ”选项视图上传操作的状态。

## 重要注意事项 {#important-notices}

**文件夹上传。** 建议使用新的“文件夹上传”功能将更大、层次化的文件夹上传到AEM中，而不是使用从Finder/资源管理器级别复制／拖放到已装载的AEM存储库中。 使用文件夹上传功能时，桌面应用程序会直接与AEM通信，因此可以更好地控制整个过程。

**使AEM会话可用。** AEM桌面应用程序依赖于打开到AEM Assets服务器的会话，以确保正确操作。 对于每天使用桌面应用程序的用户，建议在每天结束时卸载AEM资产以强制注销，然后在上午“装载AEM资产”以确保他们已登录并且网络共享可运行。

**在Finder中关闭“图标预览”。** 为了使用Finder实现对大型文件夹的高性能浏览，特别是在网络连接性差的情况下，请确保同时关闭“图标”和“图标预览”。 否则，Finder将开始下载文件夹中的每个资产以生成一个小的预览，这会导致性能下降和高带宽利用率(CQ-4219779)

* 在查找器中，转到AEM资产共享网络文件夹
* 右键单击DAM安装点
* 选择显示视图选项
* 取消选择“显示图标预览”
* 单击“用作默认值”

**连接到新AEM服务器时清理缓存。** 如果桌面应用程序连接到具有相同URL的其他AEM服务器，则不会自动清除缓存。 手动清除缓存，以确保正确操作。 请注意，测试中通常会发生这种情况，因为在同一URL上运行时，AEM安装可以被替换(CQ-4216982)

**使用CA签名的SSL证书。** 请注意，通过HTTPS安全连接连接到AEM时，AEM桌面应用程序不支持自签名SSL证书。 对于此类连接，服务器上需要CA签名的证书。 (CQ-87941)

## 已知问题 {#known-issues}

* 常规:
   * 服务器URL必须指向没有路径的服务器(例如， `http://server`、 `https://server`、 `http://server:port`或 `https://server:port`)。 不支持/content/dam以外的上下文路径和子文件夹(CQ-89343、CQ-87272)
* 文件名/本地化:
   * 未正确处理带有保留字符的文件和文件夹名称。 确保使用符合AEM要求的文件和文件夹名称(CQ-93361、CQ-93308、CQ-89276、CQ-4217183)
   * 某些应用程序（如Adobe Illustrator）可能会创建名称在AEM中不受支持的文件。 例如，在转换 `Converted` 文件后添加，这会阻止其上传(CQ-4216985)
   * 具有国际名称的资源可能每隔几秒钟出现并消失
* 登记和注销：
   * 一个用户签出的资产无法通过触屏UI中的打开操作或直接在桌面上为其他用户打开。 某些应用程序可能报告其已锁定，但在尝试打开时也会损坏甚至挂起。 (CQ-4199234)
   * 同时由多个用户更改文件可能会导致某些修改丢失。 解决方法是使用登记和注销功能来防止多个用户更改同一文件(CQ-97035)
   * 某些应用程序不正确支持只读标志，该标志允许用户保存其他用户签出的文件。 在其他用户签入文件之前，不会传输修改过的文件。 在AEM中，这两种修改都可作为资产的不同版本(CQ-89551、CQ-87572、CQ-89615)使用
   * 在Finder中独立报告检出和只读状态。 当用户注销资产时，这会生成2个锁图标(CQ-89507)
* Finder集成：
   * 拖放大文件时，当文件在后台传输时，Finder可能会超时。 这会导致 `Error - 36`。 解决方法是再次拖放或打开资产(CQ-4219628)
   * 手动重新加载文件夹并不总是有效。 解决方法： 等待30秒，以便自动更新文件夹。 (CQ-97389)
   * 更多资产信息……仅限于单个文件选择(CQ-89542、CQ-87656)
   * 在AEM资产中打开……仅限于单个文件和文件夹选择(CQ-83382)
   * 重命名没有扩展名的资源时出错(CQ-4218971)
* 复制／粘贴功能：当尚未将任何资产复制到剪贴板时，粘贴可用
* Windows:
   * 只有NTFS上完全支持具有备用数据流(ADS)的文件。 将这些文件复制到桌面应用程序提供的WebDAV共享中将导致出现警告对话框，警告用户文件具有无法复制到新位置的属性。 这通常很正常，因为这些属性仅与用户桌面上的特定应用程序相关，与实际文件内容(CQ-103770)无关(Win)
   * Windows上的桌面应用程序需要由将使用它的用户安装(CQ-4216389)(win)
   * 在断开连接后恢复批量上传后，在某些情况下，在失败的上传上单击“重试”按钮时，应用程序可能会崩溃(CQ-4251884)(Win)
